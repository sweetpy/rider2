Rider-Console — Project documentation

This `docs/` folder contains onboarding and handoff documentation for the Rider-Console web application.

Purpose
- Give a new developer or operations engineer a quick orientation.
- Describe how to get the project running locally, database import, architecture, and key files.

Files in this folder
- `Setup.md` — Quick-start and environment setup.
- `Architecture.md` — High-level architecture and important design notes.
- `Directory-Reference.md` — What each folder/file does.
- `Database.md` — How to import the schema and create initial admin users.
- `Handoff-Checklist.md` — Concrete steps for a handoff to a new team.

Where to start
1. Read `Setup.md` to get the app running locally.
2. Read `Database.md` to import the SQL schema and create admin user(s).
3. Read `Architecture.md` and `Directory-Reference.md` for code orientation.

If anything in these docs is out-of-date, update the corresponding file under `docs/` and open a PR with the change.
